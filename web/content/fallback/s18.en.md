> **Harness Layer**: Production — quality assurance for agent responses.

---

## The Problem

Your agent works. It calls tools, manages context, recovers from errors. But how do you know if its responses are actually good?

You could read every response yourself. That doesn't scale. You could ask users to rate responses. Most won't. You could A/B test against a baseline. That's slow and expensive.

You need an automated way to score response quality across multiple dimensions — coherence, relevance, groundedness — without human review for every output.

---

## The Solution

Microsoft.Extensions.AI provides a composable evaluation framework. Each evaluator scores one dimension of quality. A `CompositeEvaluator` aggregates them into a single report.

```csharp
IEvaluator coherenceEval = new CoherenceEvaluator();
IEvaluator relevanceEval = new RelevanceEvaluator();
IEvaluator fluencyEval = new FluencyEvaluator();
IEvaluator compositeEval = new CompositeEvaluator(
    coherenceEval, relevanceEval, fluencyEval);
```

Evaluators are `IEvaluator` implementations. They compose like middleware — add, remove, or replace evaluators without changing the rest of the pipeline.

## How It Works

**Step 1**: Create an IChatClient for the evaluation model.

```csharp
IChatClient chatClient = new OpenAIClient(
    new ApiKeyCredential(apiKey),
    new OpenAIClientOptions { Endpoint = new Uri(baseUrl) })
    .GetChatClient(modelId).AsIChatClient();
```

The evaluation framework uses an LLM to judge response quality. You can use the same model as your agent or a different one (e.g., a stronger model for more reliable scoring).

**Step 2**: Create evaluators.

```csharp
IEvaluator coherenceEval = new CoherenceEvaluator();
IEvaluator relevanceEval = new RelevanceEvaluator();
IEvaluator fluencyEval = new FluencyEvaluator();
IEvaluator compositeEval = new CompositeEvaluator(
    coherenceEval, relevanceEval, fluencyEval);
```

Each evaluator focuses on one aspect:

| Evaluator | What it measures |
|-----------|-----------------|
| `CoherenceEvaluator` | Is the response logically consistent? No contradictions? |
| `RelevanceEvaluator` | Does the response address the actual question? |
| `FluencyEvaluator` | Is the response well-written and natural? |
| `GroundednessEvaluator` | Is the response supported by provided context? |
| `CompletenessEvaluator` | Does the response cover all aspects of the question? |

**Step 3**: Wrap the chat client in a ChatConfiguration.

```csharp
var evalConfig = new ChatConfiguration(chatClient);
```

`ChatConfiguration` is the evaluation pipeline's connection to the LLM. It wraps `IChatClient` and adds evaluation-specific metadata.

**Step 4**: Generate a response and evaluate it.

```csharp
var messages = new List<ChatMessage> {
    new(ChatRole.User, "What is machine learning?")
};
ChatResponse response = await chatClient.GetResponseAsync(messages);

EvaluationResult result = await compositeEval.EvaluateAsync(
    messages, response, evalConfig);
```

The evaluator receives the original question, the response, and the chat configuration. It returns an `EvaluationResult` containing named metrics.

**Step 5**: Read the scores.

```csharp
var coherence = result.Get<NumericMetric>(
    CoherenceEvaluator.CoherenceMetricName);
var relevance = result.Get<NumericMetric>(
    RelevanceEvaluator.RelevanceMetricName);

Console.WriteLine($"Coherence: {coherence.Value:F1}/5 ({coherence.Interpretation?.Rating})");
Console.WriteLine($"Relevance: {relevance.Value:F1}/5 ({relevance.Interpretation?.Rating})");
```

Each `NumericMetric` has a value (typically 1-5), an interpretation (Good/Fair/Poor), and optional diagnostic text.

**Step 6**: Quick evaluation with string shorthand.

```csharp
var quickResult = await relevanceEval.EvaluateAsync(
    "What is C#?",
    "C# is a modern object-oriented programming language.",
    evalConfig);
```

For simple cases, you can pass question and answer as strings instead of full `ChatMessage` objects.

MEAI provides a comprehensive set of built-in evaluators:

```text
Coherence, Relevance, Fluency, Groundedness, Completeness,
```

## Try It

**Setup** (first run):

```sh
dotnet build
cp s18_evaluation/appsettings.example.json s18_evaluation/appsettings.json
# Edit appsettings.json with your API key
```

**Run**:

```sh
dotnet run --project s18_evaluation
```

What to observe:

1. Three queries are evaluated automatically
2. Each gets Coherence, Relevance, and Fluency scores (1-5 scale)
3. The quick evaluation shows a single-dimension score
4. Groundedness evaluation checks if the answer is supported by context

Try changing the queries or the model to see how scores change.

<details>
<summary>Dive into Evaluation Design</summary>

> The evaluation framework is built on the `IEvaluator` interface. Each evaluator is a standalone unit that can be composed, replaced, or extended.

<details>
<summary>1. Evaluators use an LLM as judge</summary>

The evaluation model (configured via `ChatConfiguration`) generates the quality scores. This means evaluation quality depends on the judge model's capability. Using a stronger model (e.g., GPT-4) to judge a weaker model's output (e.g., GPT-3.5) often gives more reliable scores than self-evaluation.

</details>

<details>
<summary>2. Composite scoring with weights</summary>

`CompositeEvaluator` aggregates individual evaluator scores. By default it uses equal weights, but you can customize weights to prioritize dimensions that matter most for your use case. For example, in a code review agent, `Groundedness` might matter more than `Fluency`.

</details>

<details>
<summary>3. Evaluation as observability signal</summary>

Evaluation scores are structured data. Log them, aggregate them, alert on them. A drop in coherence scores across many requests may indicate model degradation or a prompt regression. This makes evaluation a first-class observability signal alongside traces, metrics, and logs.

</details>

**In one sentence**: Evaluation turns "this response looks good" into "this response scores 4.2/5 on coherence" — measurable, trackable, improvable.

</details>
