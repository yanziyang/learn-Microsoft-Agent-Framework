---

## 问题

你的 Agent 在本地能工作。你用 `dotnet run` 运行它，处理一个请求，然后退出。但生产环境的 Agent 需要：

- 接受来自客户端的 HTTP 请求
- 优雅地启动和关闭
- 报告健康状态
- 发射跟踪数据以便调试请求过程中发生了什么
- 可重启、可扩展、可监控

将 Agent 作为控制台应用运行在开发阶段没问题。生产环境需要基础设施。

---

## 解决方案

ASP.NET Core 的托管模型提供了 Agent 需要的一切。`AddAIAgent()` 将 Agent 注册为一等服务。`UseOpenTelemetry()` 在每次 LLM 调用周围添加分布式跟踪。

```csharp
var builder = WebApplication.CreateBuilder(args);

// 注册带中间件的 IChatClient
builder.Services.AddSingleton<IChatClient>(instrumentedClient);

// 注册 Agent
builder.AddAIAgent("assistant",
    instructions: "You are a helpful assistant.");

var app = builder.Build();
app.MapPost("/chat", async (string message, IChatClient client) => { ... });
app.Run();
```

Agent 参与与其他托管服务相同的生命周期、健康检查和配置基础设施。

## 工作原理

**第 1 步**：设置 OpenTelemetry 跟踪。

```csharp
var serviceName = "AgentTutorial";
var sourceName = "AgentTracing";
using var tracerProvider = Sdk.CreateTracerProviderBuilder()
    .SetResourceBuilder(ResourceBuilder.CreateDefault()
        .AddService(serviceName))
    .AddSource(sourceName)
    .AddHttpClientInstrumentation()
    .AddOtlpExporter()
    .Build();
```

这创建了一个跟踪提供者：
- 将服务命名为 "AgentTutorial"
- 监听来自 "AgentTracing" 的活动
- 自动检测 HTTP 客户端调用
- 通过 OTLP 导出跟踪（兼容 Jaeger、Zipkin、Application Insights）

**第 2 步**：创建并检测 IChatClient。

```csharp
IChatClient baseClient = new OpenAIClient(
    new ApiKeyCredential(apiKey),
    new OpenAIClientOptions { Endpoint = new Uri(baseUrl) })
    .GetChatClient(modelId).AsIChatClient();

var instrumentedClient = baseClient.AsBuilder()
    .UseFunctionInvocation()
    .UseOpenTelemetry(sourceName: sourceName,
        configure: c => c.EnableSensitiveData = true)
    .Build();
```

中间件管道用以下组件包装基础客户端：
- `UseFunctionInvocation()` — 自动工具分发（见 s04）
- `UseOpenTelemetry()` — 每次 LLM 调用的跟踪 span

通过 `instrumentedClient` 的每个请求都会自动发射跟踪数据。

**第 3 步**：注册服务和 Agent。

```csharp
builder.Services.AddSingleton<IChatClient>(instrumentedClient);
builder.AddAIAgent("assistant",
    instructions: "You are a helpful assistant.");
```

`AddAIAgent()` 在 DI 容器中注册 Agent。它可以在控制器、最小 API 或其他服务中通过依赖注入使用。

**第 4 步**：定义 API 端点。

```csharp
app.MapPost("/chat", async (string message, IChatClient client) =>
{
    using var activity = new ActivitySource(sourceName)
        .StartActivity("ManualChat");
    var response = await client.GetResponseAsync(message);
    return Results.Ok(new { response = response.Text });
});

app.MapGet("/health", () =>
    Results.Ok(new { status = "healthy",
        timestamp = DateTime.UtcNow }));
```

两个端点：
- `POST /chat` — 接受消息，通过 Agent 处理，返回响应
- `GET /health` — 为负载均衡器和编排器返回健康状态

**第 5 步**：作为托管服务运行。

```sh
dotnet run --project s19_hosting_observability
```

Agent 现在：
- 监听 HTTP 请求
- 为每次 LLM 调用发射 OpenTelemetry 跟踪
- 报告健康状态
- 在收到 SIGTERM 时优雅关闭

## OpenTelemetry 给你什么

通过检测客户端的每次 LLM 调用自动产生跟踪 span：

| Span 属性 | 捕获什么 |
|-----------|---------|
| `gen_ai.system` | Provider 名称（openai 等） |
| `gen_ai.request.model` | 请求的模型 ID |
| `gen_ai.response.finish_reason` | 模型停止的原因 |
| `gen_ai.usage.prompt_tokens` | 输入 token 数 |
| `gen_ai.usage.completion_tokens` | 输出 token 数 |
| `duration_ms` | 端到端延迟 |

这些 span 流向任何 OTLP 兼容后端：Jaeger、Zipkin、Application Insights、Grafana Tempo 或任何其他可观测性平台。

<details>
<summary>深入托管设计</summary>

> 托管模型遵循 ASP.NET Core 的标准模式。Agent 只是 DI 容器中的另一个服务。

<details>
<summary>1. 为什么用 AddAIAgent() 而不是手动注册？</summary>

`AddAIAgent()` 处理 Agent 特定的关注点：会话管理、工具注册、指令绑定和生命周期钩子。手动注册需要你自己连接所有这些。扩展方法将复杂性封装在单个调用后面。

</details>

<details>
<summary>2. OpenTelemetry 中间件 vs 手动检测</summary>

中间件方法（`UseOpenTelemetry()`）自动跟踪每次 LLM 调用而不修改应用代码。手动检测（`Activity.Start/Stop`）提供更精细的控制但需要包装每个调用点。中间件是一致性和可维护性的推荐方法。

</details>

<details>
<summary>3. ASP.NET Core 免费提供健康检查</summary>

`/health` 端点是一个最小 API 路由。在生产中，你会使用 ASP.NET Core 的健康检查中间件（`app.MapHealthChecks()`），它与 Kubernetes 存活探针、负载均衡器和监控仪表板集成。Agent 的健康取决于 LLM 客户端的可用性和任何外部依赖。

</details>

**一句话**：托管把你的 Agent 从脚本变成服务 — 可观测、可扩展、生产就绪。

</details>
