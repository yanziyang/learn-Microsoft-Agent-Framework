>
> **Harness 层**: 生产 — Agent 响应的质量保证。

---

## 问题

你的 Agent 能工作了。它调用工具、管理上下文、从错误中恢复。但你怎么知道它的响应质量到底好不好？

你可以自己读每条响应。这不规模化。你可以让用户评分响应。大多数人不会。你可以和基线做 A/B 测试。这太慢太贵。

你需要一种自动化的方式来对响应质量进行多维度打分 — 连贯性、相关性、忠实度 — 而不需要人工审查每条输出。

---

## 解决方案

Microsoft.Extensions.AI 提供了可组合的评估框架。每个评估器为一个质量维度打分。`CompositeEvaluator` 将它们聚合成一份报告。

```csharp
IEvaluator coherenceEval = new CoherenceEvaluator();
IEvaluator relevanceEval = new RelevanceEvaluator();
IEvaluator fluencyEval = new FluencyEvaluator();
IEvaluator compositeEval = new CompositeEvaluator(
    coherenceEval, relevanceEval, fluencyEval);
```

评估器是 `IEvaluator` 实现。它们像中间件一样组合 — 添加、删除或替换评估器而不改变管道的其他部分。

---

## 工作原理

**第 1 步**：为评估模型创建 IChatClient。

```csharp
IChatClient chatClient = new OpenAIClient(
    new ApiKeyCredential(apiKey),
    new OpenAIClientOptions { Endpoint = new Uri(baseUrl) })
    .GetChatClient(modelId).AsIChatClient();
```

评估框架使用 LLM 来判断响应质量。你可以使用与 Agent 相同的模型，也可以使用不同的模型（例如用更强的模型来获得更可靠的评分）。

**第 2 步**：创建评估器。

```csharp
IEvaluator coherenceEval = new CoherenceEvaluator();
IEvaluator relevanceEval = new RelevanceEvaluator();
IEvaluator fluencyEval = new FluencyEvaluator();
IEvaluator compositeEval = new CompositeEvaluator(
    coherenceEval, relevanceEval, fluencyEval);
```

每个评估器关注一个方面：

| 评估器 | 测量什么 |
|--------|---------|
| `CoherenceEvaluator` | 响应是否逻辑一致？有没有矛盾？ |
| `RelevanceEvaluator` | 响应是否针对了实际问题？ |
| `FluencyEvaluator` | 响应是否写得好、自然？ |
| `GroundednessEvaluator` | 响应是否有提供的上下文支持？ |
| `CompletenessEvaluator` | 响应是否涵盖了问题的所有方面？ |

**第 3 步**：将聊天客户端包装在 ChatConfiguration 中。

```csharp
var evalConfig = new ChatConfiguration(chatClient);
```

`ChatConfiguration` 是评估管道与 LLM 的连接。它包装 `IChatClient` 并添加评估特定的元数据。

**第 4 步**：生成响应并评估。

```csharp
var messages = new List<ChatMessage> {
    new(ChatRole.User, "什么是机器学习？")
};
ChatResponse response = await chatClient.GetResponseAsync(messages);

EvaluationResult result = await compositeEval.EvaluateAsync(
    messages, response, evalConfig);
```

评估器接收原始问题、响应和聊天配置。它返回一个包含命名指标的 `EvaluationResult`。

**第 5 步**：读取分数。

```csharp
var coherence = result.Get<NumericMetric>(
    CoherenceEvaluator.CoherenceMetricName);
var relevance = result.Get<NumericMetric>(
    RelevanceEvaluator.RelevanceMetricName);

Console.WriteLine($"连贯性: {coherence.Value:F1}/5 ({coherence.Interpretation?.Rating})");
Console.WriteLine($"相关性: {relevance.Value:F1}/5 ({relevance.Interpretation?.Rating})");
```

每个 `NumericMetric` 有一个值（通常 1-5）、一个解释（Good/Fair/Poor）和可选的诊断文本。

**第 6 步**：用字符串简写快速评估。

```csharp
var quickResult = await relevanceEval.EvaluateAsync(
    "什么是 C#？",
    "C# 是一种现代面向对象的编程语言。",
    evalConfig);
```

对于简单情况，你可以用字符串代替完整的 `ChatMessage` 对象传递问题和答案。

## 可用评估器

MEAI 提供了一套全面的内置评估器：

```text
Coherence, Relevance, Fluency, Groundedness, Completeness,
Equivalence, IntentResolution, TaskAdherence, Retrieval, ToolCallAccuracy
```

每个都是一个 `IEvaluator`，为特定的质量维度打分。自由组合来创建对你的用例重要的评估配置。

## 试一下

**准备**（首次运行）：

```sh
dotnet build
cp s18_evaluation/appsettings.example.json s18_evaluation/appsettings.json
# 编辑 appsettings.json，填入你的 API key
```

**运行**：

```sh
dotnet run --project s18_evaluation
```

观察重点：

1. 三个查询被自动评估
2. 每个获得连贯性、相关性和流畅性分数（1-5 分制）
3. 快速评估显示单维度分数
4. 忠实度评估检查答案是否有上下文支持

试试修改查询或模型，看看分数如何变化。

## 接下来

评估告诉你响应的质量。但在生产环境中，你还需要将 Agent 作为服务托管、跟踪执行过程、监控健康状态。

→ s19 托管与可观测性：使用 `AddAIAgent()` 的 ASP.NET Core 托管和 OpenTelemetry 跟踪。

<details>
<summary>深入评估设计</summary>

> 评估框架建立在 `IEvaluator` 接口之上。每个评估器是独立的单元，可以组合、替换或扩展。

<details>
<summary>1. 评估器使用 LLM 作为裁判</summary>

评估模型（通过 `ChatConfiguration` 配置）生成质量分数。这意味着评估质量取决于裁判模型的能力。使用更强的模型（如 GPT-4）来判断更弱模型的输出（如 GPT-3.5）通常比自我评估给出更可靠的分数。

</details>

<details>
<summary>2. 带权重的复合评分</summary>

`CompositeEvaluator` 聚合各个评估器的分数。默认使用等权重，但你可以自定义权重来优先考虑对你的用例最重要的维度。例如，在代码审查 Agent 中，`Groundedness` 可能比 `Fluency` 更重要。

</details>

<details>
<summary>3. 评估作为可观测性信号</summary>

评估分数是结构化数据。记录它们、聚合它们、对它们设置告警。大量请求中连贯性分数的下降可能表明模型退化或提示词回归。这使得评估成为与跟踪、指标和日志并列的一等可观测性信号。

</details>

**一句话**：评估把"这个响应看起来不错"变成"这个响应在连贯性上得分 4.2/5" — 可度量、可跟踪、可改进。

</details>
